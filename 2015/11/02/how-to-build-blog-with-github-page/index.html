使用Github Pages & hexo搭建独立博客

发表于 2015-11-02

创建仓库

在github创建一个名为“.github.io”的仓库，例如我的github用户名是observer17，所以仓库名是observer17.github.io。这样就可以使用Github Pages提供的服务。

购买域名

在namecheap上购买域名

绑定独立域名到Github Pages

Github Pages会为之前创建的仓库提供一个和仓库名同名的域名服务，在仓库中加入index.html文件就可以正常访问了。要将其绑定到独立域名下，只需要在仓库根目录下新建一个CNAME文件，并写入要绑定的域名。

# my CNAME file
wangxiao.space

配置DNS

注册dnspod，添加域名

添加域名之后点击进去，添加需要的记录

两条A记录指向Github Pages提供的ip，可在这里查看。一条CNAME记录指向github pages空间。

配置namecheap

完成DNS的配置后，需要去namecheap修改nameserver。具体操作看这里。nameserver的域名为dnspod中两条默认NS记录的记录值。

搭建完成

到这里，利用Github Page的服务搭建的独立博客就完成了，但是这个时候我们的博客里还是空荡荡的。如果这个时候访问这blog的话，应该能看到github默认的404页面。

所以，我们还需要一个博客系统。下面就是用hexo来生成一个博客系统。

使用hexo生成博客系统

hexo是一个基于node的博客系统，所以请先确保已经安装好node环境。

$ npm install hexo-cli -g    # 安装hexo
$ npm init blog    # 初始化一个blog
$ cd blog    
$ npm install    # 安装node_moduels
$ hexo server    # 启动hexo

这个时候就可以在localhost:4000本地访问到你的博客了。

部署博客到github

安装部署到github所需的插件

$ npm install hexo-deployer-git -g

编辑博客根目录下的配置_config.yml，设置部署方式

deploy:
    type: git
    repo: "your git repo url"
    branch: [branch]
    message: [message]

然后执行部署命令

$ hexo d -g

部署成功的话就可以在你的域名下看到一个还算不错的博客了。
